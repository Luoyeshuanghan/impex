全局配置

impex.config(cfg)用于在impex渲染之前进行全局配置

# delimiters

  • Type: Array
  • Default: ["{{", "}}"]
  • Usage: 

    impex.config({delimiters:['{#','#}']})
    <div>
  Hello {# name #} !
</div>

# debug

  • Type: Boolean
  • Default: false
  • Usage: 

    impex.config({debug:true})

    开启debug模式后，impex会输出警告和调试信息

    这可能会导致性能降低 ！

全局API

impex.js 提供了很少的全局API只用于必要的操作，以此简化开发

# component ( name , model , services )

  • Arguments:
    • {String} name
    • {Object} model
    • {Array | null} services
  • Usage: 

    impex.component('x-panel',{
	$template : "<div style='color:red'>{{=CONTENT}}</div>",
	name : 'impex'
})
    <x-panel>
  Hello {{ name }} !
</x-panel>

# directive ( name , model , services )

  • Arguments:
    • {String} name
    • {Object} model
    • {Array | null} services
  • Usage: 

    impex.directive('mousedown',{
    onInit : function(){
        this.on('mousedown',this.$value);
    }
})
    <x-panel x-mousedown="alert($event)" >
  Hello {{ name }} !
</x-panel>

# service ( name , model , services )

  • Arguments:
    • {String} name
    • {Object} model
    • {Array | null} services
  • Usage: 

    impex.service('ViewHelper',{
	//any functions here...
});
impex.component('x-panel',{
	onCreate:function(viewHelper){
		this.viewHelper = viewHelper;
	}
},['ViewHelper']);

# filter ( name , model , services )

  • Arguments:
    • {String} name
    • {Object} model
    • {Array | null} services
  • Usage: 

    impex.filter('lower',{
    to:function(){
        return this.$value.toLowerCase();
    }
});
impex.component('x-panel',{
	$template : "<div style='color:red'>{{=CONTENT}}</div>",
	name : '<i>impex</i>'
})
    <x-panel>
  Hello {{ name => lower}} !
</x-panel>

# render ( element , model , services )

  • Arguments:
    • {HTMLElement} element
    • {Object} model
    • {Array | null} services
  • Usage: 

    impex.render(document.body,{
	//any props here...
});

    render方法是impex的入口，一旦该方法被执行，任何后续的全局配置都将无效

    大部分情况下，直接设置document.body为入口是一个不错的选择，但如果你需要只渲染某个组件 就最好指定到它impex.render(document.getElementById(),...)

# findAll ( name , conditions )

  • Arguments:
    • {String} name
    • {Object} conditions
  • Usage: 

    impex.findAll('x-panel',{name:'exp'})

组件

见 components

# $view

  • Type: View
  • Usage:

    视图对象表示组件对应的可视模型，这是一个抽象层，所以应该避免调用该对象的任何属性，而用函数取代

# $name

  • Type: String
  • Usage:

    如果是注册组件，该属性就是组件的注册名。如果是匿名组件，该属性为空

# $template

  • Type: String
  • Usage:

    模版字符串，用于定义组件的视图。使用标准HTML语法

# $templateURL

  • Type: String
  • Usage:

    模版URL，用于加载远程视图模版。首次加载是异步

# $restrict

  • Type: Object
  • Usage:

    组件约束，用于定义组件的使用范围包括上级组件限制等

    impex.component('x-panel',{
	parents:'x-window',
	children:'x-block,x-cell'
})

# on ( type , exp , handler )

  • Arguments:
    • {String} type
    • {String} exp
    • {Function} handler
  • Usage:

    绑定事件到组件

# off ( type , exp )

  • Arguments:
    • {String} type
    • {String} exp
  • Usage:

    从组件解绑事件

# find ( name , conditions , recur )

  • Arguments:
    • {String} name
    • {Object} conditions
    • {Boolean} recur
  • Usage:

    查找子组件，并返回符合条件的第一个实例

    如果不开启递归查找，该方法只会查询直接子节点集合！

# findD ( name , conditions )

  • Arguments:
    • {String} name
    • {Object} conditions
  • Usage:

    查找组件内指令，并返回符合条件的第一个实例

# watch ( expPath , cbk )

  • Arguments:
    • {String} expPath
    • {Function} cbk
  • Usage:

    监控当前组件中的模型属性变化，如果发生变化，会触发回调

    impex.component('x-panel',{
	onCreate:function(){
		this.watch('a.b.c',function(type,newVal,oldVal){
			//type could be add/delete/update
		})
	}
})

# add ( child )

  • Arguments:
    • {Component} child
  • Usage:

    添加子组件到父组件

# createSubComponentOf ( type , target )

  • Arguments:
    • {String} type
    • {View} target
  • Usage:

    创建一个未初始化的子组件

# createSubComponent ( tmpl , target )

  • Arguments:
    • {String | View} tmpl
    • {View} target
  • Usage:

    创建一个匿名子组件

# init ( )

  • Usage:

    初始化组件，该操作会生成用于显示的所有相关数据，包括表达式等，以做好显示准备

    多次初始化会导致组件错误

# display ( )

  • Usage:

    显示组件到视图上，组件必须已经初始化完成

# suspend ( hook )

  • Arguments:
    • {Boolean} hook
  • Usage:

    挂起组件，组件视图会从文档流中脱离，组件模型会从组件树中脱离，组件模型不再响应数据变化，但数据都不会销毁

    hook 标识是否保留视图占位符，如果为true，再次调用display时，可以在原位置还原组件， * 如果为false，则需要注入viewManager，手动插入视图

# destroy ( )

  • Usage:

    销毁组件，会销毁组件模型，以及对应视图，以及子组件的模型和视图

# data ( path , [val] )

  • Arguments:
    • {String} path
    • {*} [val]
  • Usage:

    设置或者获取模型值，如果第二个参数为空就是获取模型值。设置模型值时，context为当前域的模型，如果当前模型不匹配表达式，则赋值无效。获取模型值时，会从当前域向上查找，直到找到匹配对象，如果都没找到返回null

# closest ( path )

  • Arguments:
    • {String} path
  • Usage:

    查找拥有指定属性的最近的祖先组件

组件 - 生命周期

# onCreate ( [...args] )

  • Arguments:
    • {...Service} [...args]
  • Usage:

    当组件被创建时，该回调触发，系统会把指定的服务注入到参数中

# onInit ( tmplStr )

  • Arguments:
    • {String} tmplStr
  • Usage:

    当组件初始化时，该回调被触发，系统会扫描组件中的所有表达式并建立数据模型

# onDisplay ( )

  • Usage:

    当组件被显示时，该回调被触发，此时组件以及完成数据构建和绑定

# onDestroy ( )

  • Usage:

    当组件被销毁时，该回调被触发

# onSuspend ( )

  • Usage:

    当组件被挂起时，该回调被触发

组件 - 视图

# elements

  • Type: Array
  • Usage:

    当组件被创建时，该回调触发，系统会把指定的服务注入到参数中

# clone ( )

  • Usage:

    复制当前视图

# show ( )

  • Usage:

    显示视图

# hide ( )

  • Usage:

    隐藏视图

# style ( name , value )

  • Arguments:
    • {String} name
    • {*} value
  • Usage:

    获取或设置视图的样式

# attr ( name , value )

  • Arguments:
    • {String} name
    • {*} value
  • Usage:

    获取或设置视图的属性值

# removeAttr ( name )

  • Arguments:
    • {String} name
  • Usage:

    删除视图属性

# hasClass ( name )

  • Arguments:
    • {String} name
  • Usage:

    视图是否包含指定样式

# addClass ( names )

  • Arguments:
    • {String} names 空格分隔多个样式名
  • Usage:

    添加样式到视图

# removeClass ( name )

  • Arguments:
    • {String} name
  • Usage:

    从视图删除指定样式

# toggleClass ( name )

  • Arguments:
    • {String} name
  • Usage:

    从视图添加或移除指定样式

指令

# $value

  • Type: String
  • Usage:

    指令的字面值

# $name

  • Type: String
  • Usage:

    指令名称

# $params

  • Type: Array
  • Usage:

    参数列表

# $filter

  • Type: Function
  • Usage:

    过滤函数

# $final

  • Type: Boolean
  • Default: false
  • Usage:

    是否终结。终结指令会告诉扫描器不对该组件的内部进行扫描，包括表达式，指令，子组件都不会生成

    该属性与$endTag冲突，并会优先匹配

# $endTag

  • Type: String
  • Usage:

    范围结束标记，用来标识一个范围指令的终结属性名。如果设置了该标识，那么从当前指令开始到结束标识结束形成的范围，扫描器都不对内部进行扫描，包括表达式，指令，子组件都不会生成

# observe

  • Type: Function
  • Usage:

    当指令表达式中对应模型的值发生变更时触发，回调参数为表达式计算结果

指令 - 生命周期

指令继承自组件，所以指令也具有组件的生命周期。唯一例外的是，指令没有onDisplay

指令 - 内建指令

为了保持灵活性，impex.js 只是内置了有限的常用指令

# x-model

  • Usage: 

    <input x-model="exp">

    视图到模型的绑定，当控件修改值后，模型值也会修改

# x-bind

  • Usage: 

    <a x-bind:href:title.validate="address">Hello impex !</a>

    绑定视图属性，并用表达式的值设置属性。可以同时绑定多个属性

# x-show

  • Usage: 

    <div x-show="exp"></div>

    控制视图显示指令，根据表达式计算结果控制。基于CSS

# x-show-start

  • Usage: 

    
<div x-show-start="exp">start</div>
<div x-show-end>end</div>

    x-show的范围版本

# x-if

  • Usage: 

    <div x-if="exp"></div>

    效果与show相同，但是会移除视图

# x-if-start

  • Usage: 

    
<div x-if-start="exp">start</div>
<div x-if-end>end</div>

    x-if的范围版本

# x-cloak

  • Usage: 

    .x-cloak{display: none;}
    <div x-cloak class="x-cloak">你看不见我...</div>

    用于屏蔽视图初始时的表达式原始样式，需要配合class使用

# x-each

  • Usage: 

    <div x-each="collection as k => v"></div>

    each指令用于根据数据源，动态生成列表视图。数据源可以是数组或者对象/常量或者变量

# x-each-start

  • Usage: 

    
<div x-each-start="collection as k => v">start</div>
<div x-each-end>end</div>

    x-each的范围版本

过滤器

指令继承自组件，所以指令也具有组件的生命周期。唯一例外的是，指令没有onDisplay

# $component

  • Type: Component
  • Usage:

    所在组件/常量或者变量

# $value

  • Type: String
  • Usage:

    系统自动计算的表达式结果

# to ( ...args )

  • Arguments:
    • {...String} ...args 可变参数，根据表达式中书写的参数决定
  • Usage:

    转变函数，该函数是实际进行转变的入口

过滤器 - 生命周期

# onCreate ( [...args] )

  • Arguments:
    • {...Service} [...args]
  • Usage:

    当转换器被创建时，该回调触发，系统会把指定的服务注入到参数中

服务

# $singleton

  • Type: Boolean
  • Usage:

    服务是否为单例模式，如果为true，该服务就不会在注入时被实例化，比如纯函数服务

# $host

  • Type: Component | Directive | Converter | Service
  • Usage:

    服务被注入到的宿主，可能是组件，指令或者另一个服务

服务 - 生命周期

# onCreate ( [...args] )

  • Arguments:
    • {...Service} [...args]
  • Usage:

    当服务被创建时，该回调触发，系统会把指定的服务注入到参数中
