组件

是什么

在 impex.js 中一切皆组件，组件是impex构建UI的基本单元。 组件可以是一个按钮、输入框，也可以是一个面板、容器， 更可以是一个包含了按钮、输入框的面板、容器。 也就是说，组件构成了整个impex应用。 每个组件都包含自己独立的视图，控制器和数据， 通过控制器修改数据来驱动视图的更新。组件总是保证自己关联视图的状态更新。

上面这幅图很好的诠释了组件在构建应用中的角色。 impex的入口总是从一个组件开始，如果render的DOM元素并不是一个已经注册的组件， 那么系统会自动创建一个匿名组件，并递归解析内部的所有组件， 最终构成一个组件树，也就是完整的webUI

使用

使用组件包括了注册、视图注入两部分

注册 

//注册组件
impex.component('x-div',{//组件模型
	$template : "<div {{=BINDPROPS}}>{{delimiter}} {{=CONTENT}} {{delimiter}}</div>",//视图模版
	delimiter : '==' //模型属性
})
//{{delimiter}} 是组件视图中的表达式，表达式中引用了模型属性delimiter

<!-- 使用组件 -->
<x-div style='color:#aaa;font-size:2rem'>
	impex
</x-div>
<x-div style='color:#666;font-size:1.5rem'>
	soya
</x-div>
impex soya

注册组件必须在渲染视图模板之前执行，一旦视图中包含了所注册的组件，系统就会自动创建 一个该组件实例，并使用组件定义时的视图模板替换原始视图中的内容。

组件的注册是全局的，也就是说相同名字的组件会被覆盖

组件命名遵循W3C命名规范

约束

组件约束用来限制组件存在的作用域，比如只能存在于特定的组件中，或者只能包含特定的子组件， 就像tr只能包含td，并且td只能存在于tr样。 

//注册组件
impex.component('x-head',{
    $restrict:{
        children:'x-button'//限制了子组件
    },
    $template:"<div><x-button>Button A</x-button></div>"
});
impex.component('x-body',{
    $restrict:{
        children:'x-button'//限制了子组件
    },
    $template:"<span><x-button>Button B</x-button></span>"
});
impex.component('x-button',{
    $restrict:{
        parents:'x-body'//限制了父组件
    },
    $template:"<button >{{=CONTENT}}</button>"
});


<x-head></x-head><!-- 不会渲染 -->
<x-body></x-body>

模板表达式

在组件模板中使用{{= ... }}语法就可以调用模板表达式， 帮你实现诸如标签体内容引用、属性引用等。 模板表达式只做编译前的替换，没有任何其他操作，比如监控之类。更多模板 表达式语法请看这里

匿名组件

即使没有注册过组件，当调用impex.render(...)时， 系统依然会自动创建一个匿名组件。匿名组件通常由系统自动维护， 使用者不能也不应该对匿名组件进行操作。会产生匿名组件的操作包括：

  • impex.render()
  • {{# }}表达式
  • 某些指令，比如x-each产生的每个项
  • ...

属性

impex.js 中的组件表现为一个自定义标签，和其他HTML标签样，组件上也可以书写属性， 但不一定会出现在最终的结果中，这取决于组件的视图模板

自动绑定 

//注册组件
impex.component('x-div',{
	$template : "<div style='{{=style}}' ></div>"
})

<!-- 使用组件 -->
<x-div style='color:#aaa;font-size:2rem' class="skin">
	impex
</x-div>

使用模板表达式可以在视图模板中直接引用组件属性，只需要{{= 属性名 }}即可。

使用组件时，并不是书写的每个属性都会绑定到视图模板，这取决于实际的绑定表达式。

BINDPROPS 

//注册组件
impex.component('x-div',{
	$template : "<div {{=BINDPROPS}}></div>"
})

BINDPROPS 是一个内置的模板表达式常量，用来绑定组件使用时书写的所有属性。只能使用在属性中

模型属性

组件书写时的所有属性都会自动的绑定到组件实例上，并且都以字符形式。

在HTML中，标签属性是不区分大小写的<a Href="">和<a href="">没有任何区别，都会被识别为小写格式， 所以，在组件模型的属性中存在的是小写格式的href
impex会自动把横线格式的属性自动转为驼峰格式(Camel-Case)的组件属性，如 <x-comp my-prop="x"></x-comp> 组件就会自动拥有myProp属性

作用域

组件的作用域包括视图、和模型数据。

对于视图，就像本章开头介绍的那样，整个impex应用都对应了一棵组件树。 父组件的视图包含了子组件的视图。所以，一旦销毁父组件，子组件也会被销毁。

尽管组件拥有视图属性，并可以独立的对视图进行操作，比如隐藏、修改样式等...。 但是，除了为视图绑定事件外，其他操作最好使用指令进行，比如x-show等。 因为对组件来说，视图只是其中的一部分。当一个视图要显示、销毁或者挂起时， 不仅视图需要变动，组件模型本身也需要变动。只对视图操作会导致组件模型和视图状态不一致， 导致错误。

对于数据，子组件的视图表达式中可以引用到父组件的模型值，而在子组件的模型中无法直接引用。 但是，如果组件想要通用并且复用，就不能假设父组件拥有某个变量。因为组件的上级组件是不可预知的。 从这点来说，组件视图中最好总是只访问组件自身的模型值。 


//注册组件
impex.component('x-a',{//组件模型
	$template : "<div>{{a}} {{=CONTENT}}</div>",//视图模版
	a : 10 //模型属性
})
.component('x-b',{//组件模型
	$template : "<div>{{a+b}} {{=CONTENT}}</div>",//视图模版
	b : 5 //模型属性
})
.component('x-c',{//组件模型
	$template : "<div>{{a+b+c}} {{=CONTENT}}</div>",//视图模版
	c : 1 //模型属性
})

<!-- 使用组件 -->
<x-a> <!-- 顶级组件 -->
	<x-b> <!-- 上级组件 -->
		<x-c> <!-- 下级组件 -->
			impex
		</x-c>
	</x-b>
</x-a>

impex

组件通信

每个组件都应该是一个拥有独立逻辑和视图的自包含的对象，不应该直接使用父/子组件的属性或者方法， 这会造成组件之间耦合，无法实现重用。为此，impex 提供了事件接口来解决这个问题

  • on()方法用于注册自定义事件监听
  • emit()方法用于触发上级组件的自定义事件，基于冒泡方式
  • broadcast()方法用于广播下级组件的自定义事件，采用递归方式

无论emit()还是broadcast()方法，当触发一次事件后， 除非handler明确返回true，否则事件不在继续触发。

除此以外，必要时可以通过直接访问组件引用来进行数据交换，这些方式包括

  • 在视图表达式中，子组件可以直接引用父组件的属性、方法
  • 子组件可以通过$parent属性来访问父组件实例
  • 父组件可以通过find()方法来查找子组件
  • 平级组件无法直接通信
