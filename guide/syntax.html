语法

impex.js 的表达式提供了模型与视图解耦的能力，而表达式是写在HTML代码中， 为了 impex.js 能够识别这些表达式，需要遵循一些简单的语法

表达式

文本

impex.js 最直接的数据绑定方式就是文本表达式 


<div id="entry">
  Hello {{ name }} !
</div>

只需要在表达式限定符中编写标准javascript(语句)语法即可，impex.js 支持标准的javascript原生语法 


<div id="entry">
  Hello {{ name + '!' }}
</div>

你可以在表达式中书写任意复杂的javascript语句代码，impex.js 会自动分析出哪些是变量(name)，哪些是常量(!)， 并对变量进行监控。但是，定义语句是不被允许的 

<!-- 错误语法 -->
<div id="entry">
  {{ var Hello = "Hello"; name + '!' }}
</div>

HTML

默认情况，表达式总是会被渲染为纯文本，即便内容包含了HTML标签。impex.js 提供了渲染 HTML的专属表达式，基本和文本表达式相同，只是开头多了一个#号 

<!-- HTML过滤器 -->
<div id="entry">
  {{# html }}
</div>

// 组件 
var debug = impex.render(document.getElementById('entry'),{
	html:'<b> HTML </b>'
});
{{# html}}

html内容中可以继续包含impex组件，这样可以动态构建复杂的嵌套结构

模板

模板表达式只能应用于组件定义中，作为组件的视图代码 

<!-- 调用代码 -->
<x-div id="entry" name="x-div" type="demo">
  inner
</x-div>

<!-- x-div 模板代码 -->
<div {{= BINDPROPS }} id="{{= id}}">
  {{= CONTENT }}
</div>

看起来模板表达式只是比普通表达式多了一个 "=" 号，但模板表达式只在创建组件前进行预编译， 并不会监控任何变量。实际上，只是一个替换。

模板表达式中包含一些固定变量，用来表示不同的替换方式

  • CONTENT :用来替换组件标签内部的所有内容，标签体
  • BINDPROPS :用来替换组件标签上的所有属性。只能用于属性上

指令

不同于组件，指令的前缀是固定的x-，并且只能以属性的形式存在于标签中， 如果组件是用于创建视图，那么指令就是用于增强视图，比如增加事件相应等

impex.js 支持指令参数和过滤回调 

x-directive[:param1[:param2:...]][.filter]

  • :param表示一个参数，指令可以有0-N个参数，这些参数会在指令创建后，保存在$params$params属性中
  • .filter表示一个过滤回调，回调只能有一个，在每次表达式变动时都会触发，回调参数就是表达式的计算结果
在所有内置指令中，只有x-bind支持参数，x-bind

each

each是指令中比较特殊的一个，因为他的特殊语法是 impex.js 中唯一不同于原生javascript的地方 

<!-- 语法1 -->
<div x-each="collection as k => v">
  {{$index}} : {{k}} : {{v}} 
</div>

<!-- 语法2 -->
<div x-each="collection as v">
  {{$index}} : {{v}} 
</div>

就像for语句样，each指令内部会形成独立的作用域，该域中你可以使用以下变量

  • $index :索引值，从0开始
  • k :自定义变量，如果collection是数组，该值和$index相等， 如果是对象，则为对象的key
  • v :自定义变量，对应索引的集合值

事件 

<button x-click="toggleSize($event,true)"> toggle size </button>

除了与原生的javascript事件语法相同外，多了一个$event变量，该变量表示原生事件对象

过滤器

过滤器为表达式提供了额外的扩展，可以让代码更具语义化 

=> filter1[:param1[:param2:...]][.filter2[:param1[:param2:...]]]...

<!-- 金额过滤 -->
{{money => currency:¥:2}}

{{money => currency:$:3}}

{{money => currency:€:4}}



//过滤器
impex.filter('currency',{
    to:function(sym,decimal){
        var pair = this.$value.split('.');
        var inte = pair[0].replace(/\s/g,'');
        var integer = '';
        for(var i=0;i<inte.length;i++){
            integer += inte[i];
            if(i%3==0 && i+1<inte.length){
                integer += ',';
            }
        }
        return sym + ' ' + integer + parseFloat('0.'+pair[1]).toFixed(decimal).substr(1);
    }
});
{{money => currency:¥:2}}
{{money => currency:$:3}}
{{money => currency:€:4}}

过滤器可以有0-N个参数，可以看到，回调参数的数量是根据实际使用时参数数量决定。 过滤器的参数使用:分割、不区分类型

链式过滤

多个过滤器可以链式调用，使用.符分割 

{{# ' HELLO IMPEX !' => trim:side.lower.cap}}

//过滤器
impex.filter('trim',{
    to:function(type){
        if(type === 'side'){
            return this.$value.replace(/^\s*|\s*$/mg,'');
        }
        return this.$value.replace(/\s/mg,'');
    }
}).filter('lower',{
    to:function(){
        return this.$value.toLowerCase();
    }
}).filter('cap',{
    to:function(){
        var cap = this.$value[0].toUpperCase();
        return (cap + this.$value.substr(1)).big();
    }
});
{{# ' HELLO IMPEX !' => trim:side.lower.cap}}
