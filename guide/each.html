列表渲染

列表是web中最常见的功能之一，而在这之前，比拼接字符串更好的方式就是js模板了，而 impex.js 在实现这些功能之外还提供了强大的列表监控和动态刷新

x-each

传统列表实现方式是这样 
//拼接字符串
var template = '';
for(var k in items){
	var item = items[k];
	template += '<div>' + item + '</div>';
}

使用 impex.js 后是这样 


<div x-each="items as item">
	{{item}}
</div>

x-each指令会根据数据源动态创建列表，并创建一个子作用域，作用域中的表达式可以使用each中定义的变量(item)。 当数据域发生变更时，列表会自动刷新 


<ul>
	<li x-each="numbers as num">{{num}}</li>
</ul>
<button x-click="addNum()">add</button>
<button x-click="subNum()">sub</button>

impex.render(document.getElementById('entry'),{
    numbers:[1,2,3,4],
    addNum:function(){
        this.numbers.push(this.numbers.length+1);
    },
    subNum:function(){
        this.numbers.pop();
    }
});
  • {{num}}

x-each-start

x-each可以满足大部分列表需求，但有些复合列表项，比如每2个列表项输出一个分隔符， 这种需求就无法满足了。当然，impex.js 提供了列表组指令 


<ul>
	<li x-each-start="numbers as num">{{num}}</li>
	<li x-each-end x-if="num%2===0">我是分隔符</li>
</ul>
  • {{num}}
  • 我是分隔符

对象

each的数据源可以是数组也可以是对象 


<ul>
	<li x-each="obj as k => v">{{k}} : {{v}}</li>
</ul>

impex.render(document.getElementById('entry'),{
    obj:{
        name:'white',
        color:'red',
        shape:'rect'
    }
});
  • {{k}} : {{v}}

常量 


<ul>
	<li x-each="[1,2,3,4] as i">{{i}}</li>
</ul>

性能

impex.js 对each做了很多性能优化，包括差异更新、组件池、值刷新等， 所以当你的列表很大时依然可以保持良好的性能表现。可以在源码包中的demos目录中找到相关demo
