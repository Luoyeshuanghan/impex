指令

在 impex.js 中，指令只能作为自定义属性存在于视图属性中

虽然指令继承自组件，但和组件的作用完全不同。指令的作用不是替换或生成视图， 而是增强、控制视图。正因如此，指令的使用才和组件完全不同

基础

指令的注册和组件差不多，都使用全局接口

注册 

//注册指令
impex.directive('tip',{
	onInit : function(){//鼠标悬浮时提示内容
        this.on('mouseover',"showTip()");
    },
    showTip:function(){
    	alert(this.$value);
    }
})

<!-- 使用指令 -->
<div x-tip="Hello">
	impex here !
</div>
impex here !

特性

指令继承自组件，所以组件的属性、方法指令都有，但是他们之间有很大区别

  • 指令的视图是共享的，这很容易理解：同一个标签上的所有属性的视图都是这个标签
  • 指令没有onDisplay回调，因为指令的视图已经存在
  • 指令名注册时不加前缀x-，但是使用时必须有

进阶

参数和过滤回调

指令可以带参数和过滤回调，来进行更细致的视图控制，如果需要创建一个带参数的指令， 内置的x-bind指令是一个不错的参考。如何使用x-bind ?

监控回调

监控回调是指令提供的一个接口，会在每次指令表达式中的变量发生变更时触发。这是一个十分有用的特性， 很多内置指令都使用了这个接口，比如x-if,x-show等等

内嵌表达式

不同于文本或HTML表达式，通常指令中的表达式是不需要书写大括号的。但是有的时候 可以使用大括号来完成额外的操作，比如预解析 

<!-- button需要绑定不同的click函数 -->
<button x-each="btns as i=>btn" x-click="{{btn.click+'('+i+')'}}">
    {{btn.name + i}}
</button>
//注册指令
impex.render(document.getElementById('entry'),new function(){
    this.clicks = {
        c1:function(i){
            alert('impex '+i)
        },
        c2:function(i){
            alert('soya2d '+i)
        }
    }
    this.btns = [
        {name:'Button_',click:'clicks.c1'},
        {name:'Button_',click:'clicks.c2'}
    ]
});
