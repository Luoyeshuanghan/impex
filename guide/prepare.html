准备

impex.js 不支持IE8 以下浏览器  &  使用 impex.js 会降低页面SEO能力

安装

首先你需要下载源码包，你可以访问 impex.js 的 github 地址，或者直接点击下面的按钮

impex.js

使用 impex.js 非常简单，看看 impex.js 源码包为你准备了什么 


impex
├── build
│   ├── impex.all.js 未压缩的核心代码
│   ├── impex.all.min.js 压缩后的核心代码
├── ext
│   ├── impex.ext.directives.js 扩展指令包，提供了常用事件以及视图->模型绑定指令
│   ├── impex.ext.ie8.js 如果需要兼容IE8请使用
│   ├── impex.ext.router.js impex.js 提供的默认路由方案
│   └── .... 其他扩展
├── demos
│   └── .... 很多的demo
└── src
    └── .... 源代码

通常只需要一个包就可以让 impex.js 运行 

<!-- 开发环境 -->
<script src="impex.all.js"></script>
<!-- 生产环境 -->
<script src="impex.all.min.js"></script>

如果还有额外功能需求，可以引入ext包中的其他扩展。

重构

impex.js 内置了一些常用指令，如果你对容量大小十分在意，不需要或者还需要内置其他东西，可以重新构建 impex.js。 impex.js 使用 [gulp](http://gulpjs.com) 来构建源码，而在这之前你需要安装了nodejs，然后只需要在 impex 工程目录执行如下命令

安装依赖模块 npm install

执行gulp gulp

看吧，这有多简单

格式约定

impex.js 为了提高效率，约定了一些特殊的写法

  • impex对象实例中，内置属性名通常都以$开头，因为$开头的属性不会被监控
  • 函数名不以$开头，因为函数永远不会被监控
  • 定义impex对象时传递的模型数据并不会直接被impex引用，而是复制

调试

impex.js 的调试非常简单但十分强大，只要打开浏览器控制台就行，impex.js 的调试就开始于此。 为了调试 impex.js 需要设置一个日志器logger选项 

//设置日志器
imLog.level = 4;//默认1
impex.config({logger:imLog})

为了减小核心包容量，日志器是作为扩展存在。impex提供了默认的日志器impex.ext.console.js。

当你再次运行 impex.js 时，控制台会输出警告、以及调试信息，这些信息会帮助你更好的编写 impex.js 应用程序
