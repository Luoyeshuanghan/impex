Namespace: impex

impex

impex是一个用于开发web应用的组件式开发引擎，impex可以运行在桌面或移动端 让你的web程序更好维护，更好开发。 impex的目标是让开发者基于web技术以最低的学习成本获得最大的收益，所以impex会尽量简单。 impex由组件、指令、转换器和服务这几个概念构成

component(name, model, services)

impex.all.js, line 2172

定义组件
定义的组件实质是创建了一个组件类的子类，该类的行为和属性由model属性 指定，当impex解析对应指令时，会动态创建子类实例

Name Type Description
name string

组件名，全小写，必须是ns-name格式，至少包含一个"-"
model Object

组件模型，用来定义新组件模版。
模型属性是共享的，比如数组是所有实例公用。如果模型中的某些属性不想 被表达式访问，只需要在名字前加上"$"符号
模型方法会绑定到组件原型中，以节省内存
services Array | null

需要注入的服务，服务名与注册时相同，比如['ViewManager']
Returns:
this

render(element, model)

impex.all.js, line 2226

渲染一个组件，比如

    
    ...
    impex.render(document.getElementById('entry')...)
如果DOM元素本身并不是组件,系统会创建一个虚拟组件，也就是说 impex总会从渲染一个组件作为一切的开始

Name Type Description
element HTMLElement

DOM节点，可以是组件节点
model Object

模型，用来给组件提供数据支持，如果节点本身已经是组件， 该模型所包含参数会附加到模型中
